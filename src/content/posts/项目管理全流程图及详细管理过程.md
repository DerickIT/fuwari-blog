## 项目管理流程


```mermaid
graph TD
    A[项目管理流程] --> B[启动]
    B --> C[计划]
    C --> D[执行]
    D --> E[控制]
    E --> F[结尾]
    
    B --> G[制定项目章程并确定初步范围<br>需求与目标<br>范围与成功<br>明确项目组员]
    C --> H[制定项目管理计划<br>工作计划分解<br>进度计划<br>沟通计划]
    D --> I[指导管理项目执行活动<br>资源调配<br>领导团队沟通解决进度<br>把控项目变更]
    E --> J[监控项目执行并进行整体变更控制]
    F --> K[项目收尾和结束<br>审核确认<br>评估验收项目<br>总结文档归档]
```


### 启动


```mermaid
graph TD
    A[启动] --> B[启动内容]
    A --> C[项目经理责任]
    A --> D[项目组成员责任]
    A --> E[启动阶段三个关键点]
    A --> F[启动阶段常见问题]

    B --> B1[1. 明确项目需求]
    B --> B2[2. 确定项目目标]
    B --> B3[3. 定义项目相关方的期望值]
    B --> B4[4. 描述基本的项目范围]
    B --> B5[5. 选择基本的项目组成员]
    B --> B6[6. 明确项目经理]
    B --> B7[7. 确认要交付的文档]
    B --> B8[8. 由管理层、项目经理确认]

    C --> C1[1. 与客户沟通协商，明确项目需求和所需资源等]
    C --> C2[2. 在项目实施过程中不断了解客户需求]
    C --> C3[3. 保证与项目相关方的沟通并汇报项目进程]
    C --> C4[4. 挑选项目组成员，得到项目组的支持服务]
    C --> C5[5. 项目计划过程中领导和指导项目组成员服务]
    C --> C6[6. 挑选控制项目进程，保证项目按时间计划执行]

    D --> D1[1. 参与项目计划的制定]
    D --> D2[2. 服从项目经理的指挥，执行计划分配的任务]
    D --> D3[3. 保持与项目经理的沟通]
    D --> D4[4. 配合其他小组成员的工作]
    D --> D5[5. 项目核心成员对项目经理负责，保证的完成]

    E --> E1[1. 与客户、高层间沟通，明确项目需求并获得相关支持]
    E --> E2[2. 明确项目需求和目标]
    E --> E3[3. 开会讨论，明确团队并执行相关要求]

    F --> F1[需求不明确及需要沟通不够]
    F --> F2[项目组成员的选择]
    F --> F3[为促成项目，过于乐观地分析可行性]
```


### 计划


```mermaid
graph TD
    A[2 计划] --> B[计划内容]
    A --> C[计划阶段]
    
    B --> B1[具体工作任务和执行次序风险分析]
    B --> B2[风险分析]
    B --> B3[得到相应的项目相关方批准的项目计划]
    B --> B4[工具: 甘特图, 里程碑, 网络图, 思维导图]
    
    C --> C1[管理计划 时间, 范围, 风险, 沟通等]
    C --> C2[工作任务分解]
    C --> C3[任务工期估算]
    C --> C4[时间进度安排]
    C --> C5[风险与沟通计划]
    C --> C6[项目预算计划]
    C --> C7[明确项目范围及具体任务]
    C --> C8[全面的风险识别]
    C --> C9[各关键人员的识别与沟通计划]
    
    B --> D[三个关键点]
    D --> D1[主要任务]
    D --> D2[主要成果]
    D --> D3[常见问题]
    
    D2 --> E[任务分解计划]
    D2 --> F[时间进度计划]
    D2 --> G[沟通计划]
    
    D3 --> H[对工作任务分解不充分]
    D3 --> I[风险意识不强, 没有沟通计划]
    D3 --> J[计划缺乏个人责任, 项目组没有共识认识]
    
    K[工作任务分解] --> L[1. 你的项目目标是什么?项目目标的具体任务你怎么分?]
    K --> M[2. 分解的原则需全方位, 彻底独立]
    K --> N[3. 分解的方法自上而下, 头脑风暴]
    K --> O[4. 表达的形式]
    K --> P[5. 最小粒度的特征]
    
    P --> P1[一个清晰任务的完成]
    P2[一个清晰的责任人]
    P3[时间估算工期和工作量]
    
    Q[根据项目任务的执行顺序, 时间计划及所需资源来进行分析, 制定计划] --> R[进度计划] --> S[主要工具]
    
    S --> S1[甘特图 工作分解结构WBS, 责任分配矩阵RAM]
    S --> S2[里程碑]
    S --> S3[网络图]
    
    R --> T[风险计划]
    
    T --> T1[1. 识别风险]
    T --> T2[2. 评估风险等级]
    T --> T3[3. 制定风险的应对计划]
    T --> T4[4. 制定所有计划中有可能出现的风险]
    T --> T5[5. 分析执行中可能会出现的问题]
    T --> T6[6. 与项目组员一一头脑风暴]
    T --> T7[7. 回顾以往项目中曾出现的问题三大类别: 交期, 准确, 给到评价]
    
    R --> U[沟通计划]
    
    U --> U1[沟通计划是根据项目任务的执行顺序, 时间计划及所需资源来进行分析, 制定计划]
    
    V[四个沟通渠道] --> V1[正式沟通在项目的时间节点上, 通过正式的渠道传达给项目相关人, 开展必要的沟通活动]
    
    W[如何指定项目沟通制度?]
    
    W --> W1[程序干系人]
    W --> W2[所需信息]
    W --> W3[频率]
    W --> W4[方法]
    W --> W5[责任人]
    
    X[高层管理] --> X1[项目进度, 成本, 进度, 质量及效益情况]
    X --> X2[每月]
    X --> X3[报告会议]
    X --> X4[项目经理]
    
    Y[职能管理] --> Y1[问题及目标行为]
    Y --> Y2[每周]
    Y --> Y3[报告会议]
    Y --> Y4[项目经理]
    
    Z[实施] --> Z1[视导团队]
    Z --> Z2[促进解决问题]
    Z --> Z3[和谐队伍氛围]
    Z --> Z4[保证计划所需的资源]
```


### 执行


```mermaid
graph TD
    A[3 执行] --> B[项目沟通要点]
    B --> C[项目组成员对目标达成共识]
    B --> D[相互尊重]
    B --> E[项目沟通计划、规章]
    B --> F[主动倾听]
    
    A --> G[项目人员沟通]
    G --> H[项目组内沟通]
    G --> I[高层客户沟通]
    
    H --> J[四个主要沟通需求:<br>职责、授权、协调、状态]
    H --> K[会议: 项目讨论会、<br>成员进度汇报、<br>项目例会及时、公开、<br>检讨好处]
    
    I --> L[谁, 为什么需要信息?<br>他们需要什么类型的信息?何种相近程度, 频率?<br>当你和高层及客户沟通时, 你的目标是什么?<br>采用什么样的方法完成沟通?]
    
    A --> M[项目会议沟通]
    M --> N[会前]
    M --> O[会中]
    M --> P[会后]
    
    N --> Q[明确为什么开会, 以及预期获得什么效果.]
    N --> R[考虑是否可以取消会议]
    N --> S[确定要参加的最少人数]
    N --> T[确认会议室]
    N --> U[和关键与会者就会议议题及持续时间进行沟通]
    
    O --> V[做好准备按时开始, 首先说明会议目的、会议议题]
    O --> W[每位与会者都有发言机会]
    O --> X[对会议内容进行口头总结]
    
    P --> Y[会后发布会议纪要给每位参会者]
    P --> Z[会议必须产生明确的决定]
    P --> AA[所有决定必须建立即时清单]
```


### 控制


```mermaid
graph TD
    A[4 控制] --> B[控制内容]
    B --> C1[1识别计划的偏离]
    B --> C2[2采取矫正措施, 使实际进展与计划保持一致]
    B --> C3[3接受和评估相关方的项目变更请求]
    B --> C4[4必要时重新调整项目活动]
    B --> C5[5必要时调整资源水平]
    B --> C6[6得到授权者批准后, 变更项目范围, 调整项目目标并记录]
    B --> C7[7监控项目进展, 把控项目实施]

    A --> D[监控要点]
    D --> E1[高风险任务]
    D --> E2[资源费用]
    D --> E3[项目进度]
    D --> E4[人员表现]

    A --> F[方法工具]
    F --> G[方法和工具]
    G --> G1[1. 项目进度计划表]
    G --> G2[2. 百分会议]
    G --> G3[3. 观察/检查]
    G --> G4[4. 跟踪行动计划]
    G --> G5[5. 定期反馈及报告]
    G --> H[1、投入更多的人<br>2、现有的人投入更多时间（加班）<br>3、换工作效率更高的人去做<br>4、改进方法和工具，提高效率<br>5、缩小项目范围或降低活动质量<br>需要求监控过程中发现进度延迟怎么办]

    I[客户方: 不断变化的想法与欲望] --> J[变更管理]
    K[项目团队: 成员技能与团队冲突] --> J
    L[项目优先级: 资源变化, 其他项目影响] --> J

    M[根据沟通计划, 与项目相关方进<br>行良好沟通] --> N[三个关键点]
    O[严格监控进度, 及时协调解决问<br>题] --> N
    P[重点跟踪监控高风险任务, 并采<br>取有效的防范措施] --> N

    N --> Q[执行与监控阶段]
    Q --> R[常见问题]
    R --> S1[需求变更管理不善, 导致变更频<br>繁, 项目组运作混乱]
    R --> S2[跨部门项目组成员的绩效考核与<br>激励机制不完善]
    R --> S3[项目组的沟通和跨部门协作难度<br>比较大]
```


### 收尾


```mermaid
graph TD
    A[5 收尾] --> B[收尾流程]
    B --> C[评估及验收]
    B --> D[项目总监]
    B --> E[领导审核并确认]
    B --> F[文档归档]

    G[项目结束] --> H[01财务: 评估投资回报率, 评估实际费用与计划费用]
    G --> I[02时间: 与计划的一致性]
    G --> J[03质量: 项目输出的表现水平, 客户对质量的感受]
    G --> K[04人力资源: 团队精神, 激励, 态度调查]
    G --> L[05环境: 环境因素对项目活动的影响]
    G --> M[06项目计划: 计划流程的费用评估及适当的管理技术的使用]

    N[归档文档] --> O[启动]
    N --> P[计划执行]
    N --> Q[监控]
    N --> R[收尾]

    O --> S[方案合同项目基本信息表]
    P --> T[阶段时间安排项目综合计划表等]
    Q --> U[项目会议纪要变更说明, 审核记录, 阶段性报告, 相关文献]
    R --> V[项目结果文档, 项目评审验收报告, 项目总结报告, 项目交付的其他文档]

    B --> W[收尾内容]
    W --> X[项目总结]
    W --> Y[项目收尾分类]

    X --> Z[项目总结表 (项目总结, 问题与收获, 贡献度)]
    Y --> AA[合同收尾]
    Y --> AB[行政收尾]

    AC[三个关键点] --> AD[顺利完成项目评估和验收]
    AC --> AE[项目总结, 经验总结]
    AC --> AF[完整的项目信息规定]

    AG[收尾阶段] --> AH[常见问题]
    AH --> AI[经验教训的总结不够]
    AH --> AJ[项目组成员对项目的重要性认识不足]
    AH --> AK[项目的移交]
```
